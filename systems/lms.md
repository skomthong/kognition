# Learning Management Systems (LMS) and Learning Experience Platforms (LXP)

## Definition

An **LMS** (Canvas, Moodle, Blackboard) provides centralized enrollment, content delivery, gradebook, and tracking. An **LXP** (Degreed, EdCast) provides personalized content feeds, curation, and skills pathways. Both serve as the infrastructure for digital learning delivery.

## Selection Criteria

- Use the [SECTIONS framework](sections.md) for structured media/technology selection
- Interoperability: SCORM for package/runtime, xAPI for cross-system tracking, LTI for tool integration
- Accessibility: WCAG 2.2 compliance, keyboard navigation, screen reader support
- Privacy: data minimization, NIST PF-aligned governance, consent management
- Usability: reduce friction (sludge) in enrollment, navigation, and assignment submission — poor UX creates extraneous cognitive load and reduces practice behavior

## Key Principles

- LMS is infrastructure, not pedagogy. "Content dumping" (uploading slides without retrieval practice, feedback, or interaction) wastes the platform's affordances.
- Use LMS scheduling features for spaced retrieval practice — quiz schedulers + nudges + xAPI tracking.
- Run SUS (System Usability Scale) on the LMS experience quarterly; prioritize fixes that remove friction.
- LXP adds personalization but can become opaque; transparency of recommendations matters.
