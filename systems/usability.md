# Usability

## Definition

ISO 9241-11 defines usability as effectiveness, efficiency, and satisfaction in a specified context of use. Brooke's System Usability Scale (SUS) is a widely used 10-item questionnaire for rapid global usability assessment. "Sludge" describes administrative frictions and burdens that block desired action — the opposite of a nudge.

## How To Apply It

- Run SUS on the LMS/LXP experience quarterly; prioritize fixes that remove friction in enrollment, onboarding, navigation, and assignment submission.
- Treat UX friction as a learning risk: unnecessary steps and confusing UI increase extraneous cognitive load and reduce practice behavior.
- Identify and eliminate sludge: excessive clicks, confusing menus, redundant authentication, unclear instructions.
- Usability testing with actual learners reveals problems that compliance audits miss.

## Key Principles

- Poor usability is extraneous cognitive load by another name — it wastes working memory on interface navigation instead of learning.
- Completion and practice are behaviors shaped by friction; reducing UX friction is a nudge.
- Accessibility (WCAG) and usability are related but distinct: a WCAG-compliant interface can still be unusable.
