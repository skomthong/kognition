# Visualization Tools

## Summary

Visualization tools serve different purposes at different stages of design work. The key principle: **learn the concepts first, the software second.** Tool fluency without visual literacy produces polished bad charts. Visual literacy with basic tools produces clear communication.

## Tool Landscape by Purpose

### Presentation and fast communication
| Tool | Best for | Strengths | Limitations |
|---|---|---|---|
| **PowerPoint/Keynote** | Slides, simple charts, conceptual diagrams | Ubiquitous, fast, familiar | Limited chart types, no interactivity |
| **Canva** | Infographics, social media, polished visuals | Templates, drag-and-drop, professional output | Limited data handling, template dependency |
| **Figma** | Custom visuals, UI mockups, design systems | Precise control, collaboration, reusable components | Learning curve, not data-native |

### Spreadsheet-based
| Tool | Best for | Strengths | Limitations |
|---|---|---|---|
| **Excel** | Fast business charts, pivot tables, simple dashboards | Ubiquitous, powerful data manipulation | Chart defaults are poor; requires cleanup |
| **Google Sheets** | Collaborative charts, quick sharing | Free, collaborative, good enough for most needs | Fewer chart types than Excel |

### BI and dashboard tools
| Tool | Best for | Strengths | Limitations |
|---|---|---|---|
| **Tableau** | Interactive dashboards, exploration, storytelling | Powerful, flexible, excellent visualization | Cost, learning curve |
| **Power BI** | Enterprise dashboards, Microsoft ecosystem | Integrated with Office/Azure, affordable | Less flexible than Tableau for custom visuals |
| **Looker Studio** | Google ecosystem dashboards | Free, connected to Google data sources | Limited customization |

### Statistical and analytical
| Tool | Best for | Strengths | Limitations |
|---|---|---|---|
| **R (ggplot2)** | Publication-quality charts, statistical plots, reproducible analysis | Extreme flexibility, reproducible | Steep learning curve |
| **Python (matplotlib, seaborn, plotly, altair)** | Custom analysis, interactive visualization, automation | Flexible, integrates with data pipelines | Requires coding |

### Specialized design and mapping
| Tool | Best for | Strengths | Limitations |
|---|---|---|---|
| **Miro/Mural** | Conceptual maps, affinity diagrams, workshop collaboration | Real-time collaboration, flexible canvas | Not data-connected |
| **Lucidchart/draw.io/Visio** | Flowcharts, process maps, swimlanes, technical diagrams | Precise, template-rich, standard notation | Not creative/expressive |
| **Flourish** | Narrative visuals, interactive stories, animated charts | Beautiful defaults, no-code, embeddable | Less flexible for custom analysis |

## Selection Guidance

| Your need | Start here |
|---|---|
| Quick chart for a report or email | Excel/Sheets → clean it up |
| Presentation slide | PowerPoint (assertion-evidence) or Canva |
| Interactive dashboard for stakeholders | Tableau or Power BI |
| Concept map or framework diagram | Miro, Figma, or Lucidchart |
| Process map or journey map | Lucidchart, Miro, or draw.io |
| Publication-quality statistical chart | R (ggplot2) or Python (seaborn/altair) |
| Infographic or polished visual story | Canva or Flourish |
