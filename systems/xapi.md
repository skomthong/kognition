# xAPI (Experience API)

## Definition

xAPI is an interoperability standard for capturing and sharing data about learner activity and "experience" across systems via an LRS (Learning Record Store). It uses "actor-verb-object" statements (e.g., "Jane completed scenario-3") that can be generated by any system — LMS, mobile app, simulation, job tool — and stored centrally.

## When To Use It

- Use xAPI when you need cross-system activity capture: multi-tool ecosystems, mobile learning, workplace performance tracking, simulation data
- Use xAPI to implement spaced retrieval tracking: measure practice frequency, spacing patterns, and retention outcomes across modalities
- Use xAPI for learning analytics dashboards that combine data from multiple sources

## Limitations

- Integration complexity: requires LRS infrastructure and consistent statement design
- Privacy governance: more data capture means more privacy risk — minimize fields, audit trails, NIST PF alignment
- Requires intentional data design: "what statements do we need and why?" before implementation
