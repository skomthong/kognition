# Reskilling and Upskilling at Scale

## Goal

Design organizational programs that move large numbers of people from current capabilities to new ones — driven by technology shifts, market changes, or strategic pivots. The challenge: individual learning science principles must work at population scale within organizational constraints.

## The Scale Problem

What works for 20 learners doesn't automatically work for 2,000:

| Individual design | Scale challenge |
|---|---|
| Personalized pathways | Need assessment infrastructure + adaptive systems |
| Deliberate practice with coaching | Coaching capacity doesn't scale 1:1 |
| Psychological safety | Hard to create in large, diverse, hierarchical populations |
| Transfer follow-up | Manager involvement at scale requires systemic change |
| Spaced practice | Need system-level scheduling + nudging infrastructure |

## Design Principles

### Competency-based architecture
- Define target competencies FIRST ([Competency-Based Training](competency-based-training.md))
- Assess current state: where are the gaps? What can people skip?
- Design pathways from current to target — not one-size-fits-all
- Measure competency demonstration, not course completion

### Tiered delivery model
| Tier | Audience | Delivery |
|---|---|---|
| **Awareness** | Everyone | Micro-content, communication campaigns, town halls |
| **Competence** | Role-specific groups | Structured programs with practice and assessment |
| **Mastery** | Champions, specialists | Intensive cohort programs, coaching, certification |

Not everyone needs the same depth. Segment by role and current capability.

### Manager activation
- Managers are the transfer lever at scale — without them, training decays
- Brief managers BEFORE training: what's coming, why, what to reinforce
- Give managers tools: conversation guides, observation checklists, follow-up questions
- Measure manager reinforcement, not just learner completion

### Infrastructure
- [Adaptive learning](../systems/adaptive-learning.md) for personalized pathways at scale
- [xAPI](../systems/xapi.md) for tracking across systems
- [Learning analytics](../systems/learning-analytics.md) for early warning and progress monitoring
- [Spaced practice](../techniques/instructional/spaced-practice.md) built into the system, not optional

### Longitudinal progression
- Reskilling is not a single event — it's a multi-phase journey spanning months or years. See [Longitudinal Learning Design](../methods/longitudinal-learning-design.md).
- Phase 1 (0-3 months): foundational awareness + priority skills
- Phase 2 (3-9 months): deepening competence through practice, coaching, application
- Phase 3 (9-18 months): mastery, mentoring others, advanced capability
- Build transitions between phases explicitly — the gaps between phases are where momentum dies
- Sustain with [community structures](community-learning.md) that persist across phases

### Change management
- Reskilling IS organizational change — apply [Change Management](../methods/change-management.md)
- Create urgency (why now?), build coalition (leadership + champions), demonstrate quick wins (pilot results)
- Resistance is diagnostic — investigate, don't override

## Checklist

- [ ] Target competencies defined with clear performance criteria
- [ ] Current capability assessed (not assumed)
- [ ] Tiered delivery model (not everyone gets the same thing)
- [ ] Personalized pathways for competence tier
- [ ] Manager activation plan with tools and measurement
- [ ] Spaced practice and transfer interventions built in
- [ ] Analytics infrastructure for progress monitoring
- [ ] Change management plan running in parallel
- [ ] Quick wins visible within first 90 days
- [ ] Success measured by competency demonstration + business outcomes, not completion rates
