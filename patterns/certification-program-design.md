# Certification Program Design

## Goal

Design a credential program where the certification means something — it signals verified competency, not just course completion. This requires assessment-centered design with defensible validity, clear competency standards, and genuine mastery demonstration.

## Design Principles

- **Start from the competency model** — what must a certified person be able to DO? Define competencies with observable performance criteria before designing anything else (backward design).
- **Assessment IS the architecture** — the certification exam or performance assessment defines the standard. Everything else (instruction, practice, materials) exists to prepare learners to meet that standard.
- **Separate practice from credentialing** — gamification, low-stakes retrieval practice, and collaborative learning belong in the PREPARATION layer. The credentialing decision requires controlled, valid, fair assessment.
- **Mastery-based progression** — learners advance by demonstrating competency, not by completing time. Allow flexible pacing where possible.
- **Spaced practice for maintenance** — certification should include renewal or maintenance requirements with spaced retrieval to prevent skill decay.
- **Validity and fairness** — follow testing standards. Build a validity argument. Audit for bias. Provide accommodations. Document everything.

## Checklist

- [ ] Competency model defined with observable performance criteria
- [ ] Assessment designed FIRST, instruction designed to support it
- [ ] Practice environment separated from credential decision
- [ ] Mastery-based progression (not seat-time)
- [ ] Multiple assessment methods (knowledge test + performance demonstration)
- [ ] Validity argument constructed (content validity, criterion validity, consequential validity)
- [ ] Fairness audited (DIF analysis, accommodation options, bias review)
- [ ] Maintenance/renewal requirements with spaced retrieval
- [ ] Stakeholder recognition of the credential's meaning
