# Blended Learning Design

## Goal

Combine synchronous and asynchronous, online and face-to-face, self-paced and facilitated elements into a coherent program where each modality does what it does best — rather than simply "putting some content online."

## Design Principles

- **Each modality earns its place** — don't put content online because it's cheaper; put it online because async is BETTER for that content (self-paced review, retrieval practice, video demonstration). Use sync for what sync is BETTER for (discussion, coaching, collaborative problem-solving, live practice with feedback).
- **Design the handoffs** — the transitions between modalities are where programs fail. If learners complete async pre-work but the live session doesn't reference it, they stop doing pre-work.
- **Spacing is built in** — blended programs naturally space learning over time. Leverage this by inserting retrieval checks between sessions.
- **CoI presences must be maintained across modes** — cognitive, social, and teaching presence need to function in both async and sync. An async module with no teaching presence feels abandoned.

## Modality Selection

| Modality | Best for | Worst for |
|---|---|---|
| **Async self-paced (video, reading, e-learning)** | Content acquisition, retrieval practice, reflection | Deep discussion, real-time coaching, complex practice with feedback |
| **Sync virtual (video call, webinar)** | Discussion, Q&A, short practice, guest speakers | Long lectures, content delivery (do async instead) |
| **Sync in-person** | Complex practice, simulation, relationship building, high-stakes debrief | Content delivery, individual reflection |
| **Async collaborative (forum, shared docs)** | Reflection, peer feedback, extended discussion | Urgent decisions, real-time problem-solving |

## Checklist

- [ ] Each modality justified by what it uniquely enables
- [ ] Handoffs between modalities are explicit (pre-work → live session → post-work)
- [ ] Spacing designed between sessions with retrieval checks
- [ ] Teaching presence maintained in async components
- [ ] Social interaction exists in both sync and async
- [ ] Assessment aligned to the modality where the competency is practiced
