# 4C/ID Model

## Definition

The Four-Component Instructional Design model (van Merriënboer, 1997; van Merriënboer & Kirschner) is designed for training complex cognitive skills — tasks that require the integration of knowledge, skills, and attitudes and cannot be learned by practicing component parts in isolation. It is grounded in cognitive load theory and addresses the problem of transfer from training to real performance.

## How It Works

Four interrelated components:

| Component | Purpose | Example |
|---|---|---|
| **Learning tasks** | Whole, authentic tasks of increasing complexity | Progressively harder patient cases, project scenarios, or business simulations |
| **Supportive information** | Explains how the domain is organized, how to approach problems | Mental models, systematic approaches, domain theory (provided per task class) |
| **Procedural information** | Step-by-step instructions for routine aspects | How-to guides, checklists, software tutorials (provided just-in-time) |
| **Part-task practice** | Repeated practice of routine aspects that need automation | Drill on calculations, keyboard shortcuts, diagnostic procedures |

Tasks are organized into **task classes** of increasing complexity. Within each class, tasks vary in surface features but share structural complexity. Scaffolding decreases within each class (guidance fading).

## Key Principles

- **Whole-task approach**: learners practice complete, meaningful tasks from the start — not isolated sub-skills assembled later.
- **Manage cognitive load**: supportive information reduces intrinsic load by providing schemas; procedural information is given just-in-time to avoid overload; part-task practice automates routine components to free capacity for complex reasoning.
- **Task-class progression**: complexity increases across task classes, with scaffolding faded within each class.
- **High design effort**: 4C/ID requires thorough task and skills analysis. Best suited for complex professional skills where transfer matters.

## When To Use It / When Not To

**Use when:** training complex professional skills — medical diagnosis, project management, engineering design, leadership, financial analysis — where integration and transfer are critical.

**Limitations:** high design effort and expertise in task analysis required. Less useful for simple procedural training where a linear approach suffices.
