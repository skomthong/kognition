# SAM (Successive Approximation Model)

## Definition

SAM (Allen & Sites, 2012) is an iterative instructional design process model emphasizing successive approximation and early prototyping. SAM1 is a simple three-iteration cycle (evaluate → design → develop). SAM2 adds a Preparation phase, Iterative Design phase (with "Savvy Start"), and Iterative Development phase for larger projects.

## How To Apply It

- **Savvy Start**: bring stakeholders, SMEs, and learners together early to brainstorm, prototype, and align before committing to full production.
- **Iterative design**: build → test → refine in rapid cycles rather than completing a full design document before development.
- **Iterative development**: develop → test → refine actual learning products with learner feedback at each cycle.

## When To Use It / When Not To

**Use when:** rapid iteration and stakeholder feedback are central; the problem is not fully defined upfront; time-to-learner matters more than documentation completeness.

**Limitations:** Can degrade into "iterate without principles" if not guided by learning science (CLT, retrieval practice, alignment). Requires stakeholder willingness to participate in iterative reviews. Less suited to heavily regulated contexts that require upfront documentation.
