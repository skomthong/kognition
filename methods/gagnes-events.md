# Gagné's Events of Instruction

## Definition

Gagné's nine events of instruction (1965/1985) provide a structured sequence for lesson design based on internal learning conditions — the cognitive processes that must occur for learning to happen. Each event maps to an internal process and provides external support for that process.

## How It Works

| Event | Internal process supported | Example |
|---|---|---|
| 1. **Gain attention** | Alertness, reception | Provocative question, surprising statistic, short scenario |
| 2. **Inform learner of objectives** | Expectancy | "By the end, you'll be able to..." |
| 3. **Stimulate recall of prior learning** | Retrieval of existing schemas | "What do you already know about...?" |
| 4. **Present the content** | Selective perception | Explanation, demonstration, worked example |
| 5. **Provide learning guidance** | Semantic encoding | Cues, examples, analogies, concept maps |
| 6. **Elicit performance** | Response generation | Practice activity, problem set, simulation |
| 7. **Provide feedback** | Reinforcement, error correction | Corrective feedback on practice |
| 8. **Assess performance** | Retrieval, evaluation | Quiz, performance task, rubric-based assessment |
| 9. **Enhance retention and transfer** | Retrieval, generalization | Spaced practice, varied contexts, job application |

## Key Principles

- The events are a design checklist, not a rigid script — experienced designers adapt, combine, and reorder events based on context.
- Events 3, 6, 7, and 9 align directly with retrieval practice, feedback, and spaced practice — the highest-evidence learning tactics.
- For experienced adult learners, events 2-3 can be brief (they already know why they're here); events 6-9 should dominate (practice and application).
- The events framework is most useful for structuring individual lessons or modules, not entire programs (use backward design or 4C/ID for program-level design).

## When To Use It / When Not To

**Use when:** designing structured lessons, modules, or e-learning sequences — especially when you need a reliable checklist for lesson completeness. Good for novice instructional designers learning to structure instruction.

**Limitations:** can become formulaic if applied mechanically. Does not address program-level design, transfer climate, or social learning. The framework describes what to do in a lesson, not how to design a curriculum.
