# Learner Journey Mapping

## Definition

Learner journey mapping adapts UX journey mapping to learning contexts. It maps the end-to-end learner experience across phases — from awareness through application — identifying touchpoints, emotional states, pain points, and moments of truth at each stage.

## How It Works

Typical journey phases for adult learning:

| Phase | Key questions | Common pain points |
|---|---|---|
| **Awareness** | How do learners discover the program? | Unclear value proposition, hard to find |
| **Enrollment** | How easy is sign-up? | Friction, confusing prerequisites, cost uncertainty |
| **Onboarding** | First experience with the program | Disorientation, unclear expectations, tech barriers |
| **Learning** | Core instructional experience | Cognitive overload, irrelevant content, poor pacing |
| **Practice** | Applying what was learned | No practice opportunities, weak feedback |
| **Application** | Using skills on the job | Weak transfer climate, no support, forgetting |
| **Support** | Ongoing help and community | No follow-up, isolation, skill decay |

## Key Principles

- Journey maps reveal problems that outcome alignment misses — onboarding friction, post-program transfer gaps, emotional dips.
- Different from a syllabus or curriculum map: journey maps include pre-learning and post-learning experiences, emotional arc, and system touchpoints.
- Build from real data: learner interviews, analytics, support tickets, dropout analysis — not assumptions.
- Use journey maps to identify where nudges, scaffolds, and support interventions should be placed.

## When To Use It / When Not To

**Use when:** designing or redesigning programs that span multiple touchpoints and time periods. Especially valuable for blended programs, onboarding, and certification pathways. **Limitations:** can become time-consuming and decorative if not tied to specific design decisions. Keep it actionable.
