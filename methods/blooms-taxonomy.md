# Bloom's Taxonomy

## Definition

Bloom et al. (1956) developed the Taxonomy of Educational Objectives (cognitive domain) to classify educational goals across cognitive levels: knowledge, comprehension, application, analysis, synthesis, evaluation. Anderson & Krathwohl (2001) revised it to: remember, understand, apply, analyze, evaluate, create — adding a two-dimensional matrix (cognitive process x knowledge dimension).

## How To Apply It

- Use Bloom to ensure outcomes and assessments are not overly concentrated at recall levels unless recall is the real performance requirement.
- Use as a planning tool: foundational knowledge supports higher-order application, analysis, and evaluation, but the taxonomy is a classification aid, not a rigid sequence.
- Use to diversify assessment: if all quiz items test "remember," you're not assessing what matters.

## When To Use It / When Not To

**Use when:** writing learning outcomes, auditing assessment cognitive demand, communicating with stakeholders about what "learning" means.

**Limitations:** Can be misused as a simplistic hierarchy that ignores context and domain differences. It assists classification but does not explain learning mechanisms. "Higher" is not always "better" — procedural recall matters in safety-critical contexts.

- Anderson, L. W., & Krathwohl, D. R. (Eds.) (2001). *A Taxonomy for Learning, Teaching, and Assessing*. Longman.
