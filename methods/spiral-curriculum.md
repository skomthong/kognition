# Spiral Curriculum Design

## Definition

The spiral curriculum (Bruner, 1960) revisits key topics repeatedly across a program, with each return at a higher level of complexity, depth, and sophistication. Instead of "cover it once and move on," core concepts are introduced simply, then revisited with more nuance, more application, and more integration as the learner's schema develops.

## How It Works

```
Visit 1: Introduce concept simply → foundational understanding
Visit 2: Revisit with more complexity → deeper analysis, new connections
Visit 3: Revisit with application → transfer to practice, integration with other concepts
Visit 4: Revisit with mastery challenge → edge cases, expert-level judgment
```

Each visit builds on prior knowledge, which means [spacing](../techniques/instructional/spaced-practice.md) is built into the structure — the time between visits acts as a natural spacing interval.

## Key Principles

- **Core concepts deserve multiple passes** — important ideas can't be "covered" in one session. Understanding deepens through repeated engagement at increasing levels.
- **Complexity increases, not just repetition** — each revisit must ADD something: new context, higher Bloom level, more challenging application, integration with new concepts. Simple repetition is not spiral.
- **Align with expertise development** — early visits suit novice-level instruction (rules, worked examples); later visits suit competent/proficient instruction (cases, judgment, edge cases). See [Expertise Development](../foundations/expertise-development.md).
- **Natural spacing** — spiral curriculum creates distributed practice as an architectural feature, not an add-on.
- **Builds schema progressively** — each visit adds connections to the learner's existing schema, producing the richly organized knowledge structures that characterize expertise.

## Application to Learning Design

- **Multi-module programs**: introduce cognitive load theory in Module 1 (basic concepts), revisit in Module 4 (applied to media design), revisit in Module 8 (managing complexity in simulations)
- **Certification programs**: core competencies revisited at increasing difficulty across levels
- **Leadership development**: introduce a leadership model → apply to case → apply to own situation → teach it to others
- **This knowledge base**: the same foundations (memory, attention, load) appear across multiple technique and method entries, each time in a new applied context

## Comparison with Other Sequencing

| Approach | Structure | Risk |
|---|---|---|
| **Linear** | Cover A, then B, then C. Never return. | Forgetting; no integration |
| **Spiral** | Introduce A, B, C simply → revisit A at depth, B at depth → integrate A+B+C | Requires more design effort; risk of feeling repetitive if each visit doesn't add enough |
| **Modular** | Self-contained units in any order | No guaranteed progression; no integration |
