# Thematic Analysis

## Definition

Thematic analysis (Braun & Clarke, 2006) is a method for identifying, analyzing, and reporting patterns (themes) within qualitative data — interviews, open-ended survey responses, observation notes, discussion transcripts. It is the most accessible qualitative analysis method and the most commonly used in applied research.

## How It Works

Six phases (Braun & Clarke):

| Phase | What you do | Output |
|---|---|---|
| **1. Familiarize** | Read and reread the data, noting initial ideas | Immersion, initial observations |
| **2. Generate initial codes** | Label interesting features of the data systematically | Codes (short labels applied to data segments) |
| **3. Search for themes** | Group codes into potential themes; gather all data relevant to each | Candidate themes with supporting data |
| **4. Review themes** | Check themes against the coded data AND the full dataset — refine, split, merge | Refined theme map |
| **5. Define and name themes** | Clarify what each theme IS and IS NOT; write a concise definition | Named themes with clear scope |
| **6. Write up** | Select compelling examples; tell the analytical story | Narrative report with evidence |

## Key Distinctions

### Codes vs. themes
- **Code**: a label for a specific data segment ("time pressure," "manager support," "forgot to apply")
- **Theme**: a pattern across multiple codes that captures something meaningful ("Transfer barriers are primarily environmental, not cognitive")

A theme is an INTERPRETATION, not just a category. "Time" is a category; "Adults deprioritize learning when time scarcity makes the cost calculation unfavorable" is a theme.

### Inductive vs. deductive coding
- **Inductive**: codes emerge from the data — you don't start with a framework
- **Deductive**: codes come from a pre-existing framework (e.g., coding learner interviews using COM-B categories)
- **Hybrid**: start with a framework, add inductive codes as new patterns emerge

## Key Principles

- **Themes don't "emerge" passively** — the researcher actively constructs themes through analytical judgment. Own your interpretation.
- **Rich description vs. interpretation** — decide whether you're providing a rich description of the full dataset or an in-depth analysis of specific themes. Trying to do both produces shallow work.
- **Evidence chain** — every theme must be supported by multiple data segments (quotes, observations). One quote is an anecdote, not a theme.
- **Negative cases** — look for data that DOESN'T fit your themes. These refine and strengthen the analysis.

## Application to Learning Design

- **Learner research**: code interview transcripts to identify needs, barriers, motivations
- **Program evaluation**: analyze open-ended feedback to identify patterns beyond survey scores
- **Transfer assessment**: code post-program interviews to understand application barriers
- **Case development**: analyze critical incidents for scenario and case material
- **Visualize**: use [Qualitative Visualization](../techniques/visual/qualitative-visualization.md) — affinity diagrams, thematic maps, coded matrices
