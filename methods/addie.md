# ADDIE

## Definition

ADDIE (Analyze, Design, Develop, Implement, Evaluate) is a label for a family of instructional systems design (ISD) process models, not a single authored model. Molenda argues "ADDIE" is an acronym whose specific origin is elusive — it emerged as a common label for the generic ISD process rather than from a single publication.

## How To Apply It

- **Analyze**: learner analysis, environment analysis, task analysis, performance gap diagnosis
- **Design**: learning outcomes, assessment strategy, sequencing, media selection
- **Develop**: create materials, build in LMS, produce media, pilot test
- **Implement**: deliver, facilitate, support learners
- **Evaluate**: formative (during development) and summative (after delivery); [Kirkpatrick](kirkpatrick.md) levels or [CIPP](cipp.md)

## When To Use It / When Not To

**Use when:** governance and traceability matter — regulatory, accreditation, large multi-stakeholder programs. ADDIE provides a common lifecycle language.

**Limitations:** Often misused as a rigid linear waterfall. In practice, good ADDIE is iterative. But the label can encourage "analysis paralysis" and bureaucratic documentation that delays learner contact. For rapid iteration, consider SAM or agile variants.
