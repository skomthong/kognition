# Knowledge Decay Management

## Definition

Knowledge decay is the loss of learned information over time. Ebbinghaus (1885) provided foundational experimental evidence on forgetting, showing a rapid initial decline that slows over time. Cepeda et al. quantitatively synthesize the distributed practice effect across hundreds of studies, showing robust benefits of spacing and documenting timing moderators.

## How It Works

Forgetting follows a predictable curve. Each successful retrieval flattens the curve and extends the retention interval. The practical implication: treat post-program decay as expected and design countermeasures into the program plan.

Two evidence-supported countermeasures:
- **Distributed practice (spacing)**: schedule retrieval at expanding intervals rather than massing practice in a single event
- **Retrieval practice (testing)**: use testing as a learning event, not only assessment; each retrieval strengthens the memory trace

## Key Principles

- Measure maintenance (30/60/90 day follow-ups), not only immediate post-tests, because workplace transfer includes generalization and maintenance over time.
- Boosters are not optional extras — they are architectural features of any program that targets durable performance.
- Use job-realistic cues in retrieval practice (not abstract flashcards alone) to support encoding specificity and transfer.
- Use LRS analytics (xAPI) to detect drop-off and trigger targeted refresh.
