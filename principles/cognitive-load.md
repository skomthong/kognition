# Cognitive Load Theory

## Definition

Cognitive load theory (Sweller, 1988) holds that learning can be impaired when problem solving overloads the limited processing capacity of working memory. Instruction should be designed to manage the demands placed on working memory so that learners can build and automate schemas.

## How It Works

Working memory can hold and process only a small amount of novel information at once. Long-term memory, by contrast, has effectively unlimited capacity and stores organized knowledge structures (schemas). Learning occurs when schemas are constructed in working memory and transferred to long-term memory. When instruction imposes too much simultaneous demand, schema construction fails.

Three types of cognitive load interact during learning:

- **Intrinsic load** — driven by the inherent complexity of the material and the learner's prior knowledge. Cannot be reduced without simplifying the content or building prerequisite schemas first.
- **Extraneous load** — caused by poor instructional design (redundant text, split attention, irrelevant elements). Can and should be minimized.
- **Germane load** — the productive effort devoted to schema construction and automation. Design should maximize this.

## Key Principles

- Use worked examples for novices to reduce problem-solving search and free working memory for schema building.
- Segment complex material into manageable parts.
- Eliminate split attention by integrating related text and visuals.
- Remove redundant information (redundancy effect).
- Fade scaffolding as expertise grows (expertise reversal effect — what helps novices can hinder experts).
- Scaffold heavily when using problem-based or discovery approaches with novices to avoid overload.
