# Causal Loop Diagrams and System Archetypes

## Definition

Causal loop diagrams (CLDs) are visual tools for mapping how variables in a system influence each other through feedback loops. System archetypes (Senge, 1990) are recurring CLD patterns that appear across domains — recognizing them helps diagnose systemic problems and find leverage points.

## How to Read and Build CLDs

### Notation
- **Variables**: things that can increase or decrease (completion rate, manager support, workload)
- **Arrows**: causal influence. A → B means "A influences B"
- **Polarity signs**: (+) means same direction (A increases, B increases); (-) means opposite direction (A increases, B decreases)
- **Loop labels**: R = reinforcing (amplifying); B = balancing (stabilizing)

### Example: The Transfer Decay Loop

```
Manager support (+)→ Practice on the job (+)→ Skill improvement (+)→ Performance results (+)→ Manager confidence (+)→ Manager support
                                                                                              [R: Reinforcing — virtuous cycle]

But also:
Workload (+)→ Time pressure (+)→ Shortcuts (-)→ Practice on the job
                                                [B: Balancing — competing demand]
```

When the balancing loop dominates, the virtuous cycle breaks.

## Common System Archetypes

| Archetype | Pattern | L&D example |
|---|---|---|
| **Limits to Growth** | Success feeds growth, but eventually hits a constraint that slows it | Early training adoption grows quickly, then hits capacity limits (facilitator availability, budget) |
| **Shifting the Burden** | A quick fix diverts attention from the fundamental solution | Buying more content instead of fixing the practice/feedback system |
| **Fixes That Fail** | A fix produces unintended side effects that worsen the original problem | Mandating completion → learners rush through → quality of learning drops → more training mandated |
| **Eroding Goals** | When performance falls short, the goal is lowered instead of performance raised | "80% completion target" dropped to 60% when it's not met |
| **Success to the Successful** | Resources flow to winners, starving others | Best programs get more budget; struggling programs get cut — widening the gap |
| **Tragedy of the Commons** | Individual rational behavior depletes a shared resource | Every team creates their own training → LMS becomes cluttered and unfindable |

## How to Use CLDs

1. **Diagnose**: when something isn't working, draw the CLD. Where are the reinforcing loops? Where are the balancing loops? What's dominating?
2. **Find leverage**: which variable, if changed, would shift the most loops? That's the intervention point.
3. **Communicate**: CLDs make systemic arguments visible to stakeholders who otherwise see only linear cause-effect.
4. **Anticipate**: before launching an intervention, draw the CLD. What feedback loops will your intervention trigger? What unintended consequences are plausible?
