# Structured Thinking

## Summary

Structured thinking is the ability to organize complex information into clear, logical frameworks before writing or communicating. It includes systems thinking (seeing connections and feedback loops), abstraction (moving between detail and big picture), and decomposition (breaking problems into parts). It's the thinking equivalent of [Chunking and Information Hierarchy](chunking-and-information-hierarchy.md) — applied to IDEAS rather than TEXT.

## Core Skills

### MECE decomposition (Mutually Exclusive, Collectively Exhaustive)
Break a problem into parts that don't overlap and cover everything:
- "What affects transfer?" → Trainee factors, Training design factors, Work environment factors (Baldwin & Ford) — each distinct, together complete

### Issue trees
Start with the question, branch into sub-questions, then into sub-sub-questions:
```
Why is completion low?
├── Motivation problem?
│   ├── Don't see the value?
│   └── Competing priorities?
├── Ability problem?
│   ├── Too hard?
│   └── Tech barriers?
└── Opportunity problem?
    ├── No time allocated?
    └── Manager doesn't support?
```

### Abstraction — moving between levels
- **Zoom out**: from specific observation to pattern to principle
- **Zoom in**: from principle to application to specific action
- The skill is moving between levels deliberately: "This specific observation (zoom in) illustrates a broader pattern (zoom out) that suggests this design principle (abstract) which we can apply by doing X (concrete)."

### Systems thinking
- See feedback loops: "Low completion → manager loses confidence → reduces training time → lower completion"
- See unintended consequences: "Adding gamification increased engagement but shifted motivation from intrinsic to extrinsic"
- See leverage points: where is the smallest change that produces the largest effect?

## Application to Writing

Structured thinking produces structured writing. If your thinking is a messy web, your writing will be a messy web. If you can organize your thinking into clear frameworks, the writing follows:
- MECE thinking → [pyramid structure](pyramid-principle.md)
- Issue tree thinking → logical section structure
- Abstraction → appropriate level of detail for the audience
- Systems thinking → seeing connections that become cross-references and implications

## How To Practice

1. Take a design problem and build an issue tree with 3 levels. Is each level MECE?
2. Take a specific observation and practice zooming: what pattern does it illustrate? What principle does the pattern suggest? What action does the principle recommend?
3. Map a feedback loop in a learning system you know. Where's the leverage point?
