# Pyramid Principle

## Summary

The Pyramid Principle (Barbara Minto, 1987) states that ideas in writing should form a pyramid: start with the answer/recommendation at the top, then group supporting arguments below, with details supporting each argument. The reader gets the conclusion first and supporting logic second — the opposite of "mystery" structure where the conclusion comes last.

## Why It Works (Connection to Foundations)

- [Cognitive Load](../../principles/cognitive-load.md) — leading with the conclusion gives the reader a schema to organize incoming details. Without it, every detail floats unanchored in working memory.
- [Attention](../../foundations/attention.md) — busy readers (executives, stakeholders, adult learners) decide quickly whether to keep reading. Top-line first means they get the value even if they stop.
- [Schema and Mental Models](../../principles/schema-and-mental-models.md) — the top-line activates or creates the reader's framework; details then slot into that framework.

## How It Works

### The SCR structure (Situation–Complication–Resolution)

1. **Situation** — the context the reader already knows or agrees with
2. **Complication** — the problem, tension, or question that disrupts the situation
3. **Resolution** — your answer, recommendation, or key finding (this is the pyramid's apex)

Then: group supporting arguments under the resolution, each with its own evidence.

### MECE grouping

Supporting arguments should be **Mutually Exclusive, Collectively Exhaustive** — no overlaps, no gaps. This ensures the logic is clean and complete.

### Visual structure

```
        [Main message / recommendation]
       /              |              \
  [Argument 1]   [Argument 2]   [Argument 3]
   /    \          /    \          /    \
[detail][detail] [detail][detail] [detail][detail]
```

## Application to Learning Design

- **Executive communication** — present recommendations before rationale
- **Facilitator guides** — lead with the session objective, then the how
- **Learner-facing materials** — state what you'll learn and why before diving into content
- **Stakeholder presentations** — recommendation on slide 1, evidence on slides 2-5
- **Email** — first sentence = the action needed; rest = supporting context

## Common Mistakes

- **Bottom-up writing** — presenting all the research, analysis, and details before revealing the conclusion. This works for mystery novels, not for professional communication.
- **No grouping** — listing 12 points without clustering them into 3-4 groups
- **Overlapping groups** — arguments that blur into each other (not MECE)

## How To Practice

1. Write a one-sentence answer to the question your document addresses. Put it first.
2. List your supporting points. Group them into 3-4 clusters. Label each cluster.
3. Check: are the clusters MECE? Does each cluster genuinely support the top-line?
