# Data Storytelling

## Summary

Data storytelling is the bridge between data visualization and narrative — guiding the audience from raw data to insight to action through deliberate annotation, sequencing, and framing. A chart shows data; data storytelling shows what the data MEANS.

## Core Techniques

### A. Annotated insight
Don't just show data — explain what matters:
- *Chart alone*: a line chart showing retention over 90 days
- *With annotation*: same chart + callout: "Without spacing, 62% of content is forgotten by day 30. With spacing, retention holds at 84%."

The annotation IS the story. The chart is the evidence.

### B. Guided sequence
Reveal one point at a time rather than showing everything at once:
1. Show the baseline: "Here's current performance"
2. Introduce the problem: "Here's where the gap is"
3. Show the evidence: "Here's what the research says"
4. Present the solution: "Here's what changes"
5. Project the impact: "Here's what we expect"

This maps to the [pyramid principle](../writing/pyramid-principle.md): situation → complication → resolution, supported by evidence at each step.

### C. Highlighting
Use visual contrast to direct focus:
- Gray out all data except the one series that matters
- Bold the row in a table that carries the insight
- Use a single color for the key finding, neutral for everything else

Highlighting uses [preattentive attributes](../../principles/visual-encoding.md) deliberately — color difference, size difference, isolation.

### D. Framing question
Start with the question the data answers:
- "What happened to retention after we removed spacing?"
- "Which cohort transferred skills most effectively?"
- "Where do learners drop off?"

The frame determines what the audience looks for in the data and how they interpret it.

### E. Narrative arc for data presentations
Apply story structure to data presentations:
1. **Context**: what's the situation? What does the audience already know?
2. **Tension/problem**: what's wrong? What's the gap? What's at stake?
3. **Evidence**: what does the data show? (This is where charts live)
4. **Implication**: what does the evidence mean?
5. **Action**: what should we do?

## Audience Adaptation

| Audience | Data storytelling approach |
|---|---|
| **Executives** | Top-line first, one chart per insight, clear recommendation, minimal detail |
| **Researchers** | Full data, uncertainty shown, methods transparent, nuance preserved |
| **Learners** | Scaffolded, annotated heavily, progressive reveal, reduced complexity |
| **Operations** | Real-time, threshold-based, action-oriented, drill-down available |

## Common Mistakes

- **Chart dump** — 12 charts with no narrative connecting them
- **No annotation** — the audience has to figure out the insight themselves
- **Data without "so what"** — "completion was 73%" (and? is that good? bad? compared to what?)
- **Too much data, too little story** — overwhelming detail that obscures the point
- **Misleading framing** — cherry-picking data that supports a predetermined conclusion
