# Interactive Media Patterns

## Summary

Interactive media patterns are reusable interaction types used in e-learning, simulations, and digital learning materials. Each pattern has specific affordances — what it enables the learner to DO — and specific learning applications. Choosing the right pattern depends on the learning objective, not the tool's feature list.

## Pattern Catalog

| Pattern | Learner action | Best for | Cognitive engagement level |
|---|---|---|---|
| **Drag-and-drop** | Move items to correct positions/categories | Classification, sequencing, labeling, matching | Active-Constructive (depends on complexity) |
| **Hotspot / clickable image** | Click regions to explore detail | Spatial knowledge, equipment identification, process exploration | Active |
| **Accordion / reveal** | Click to expand hidden content | Progressive disclosure, self-paced exploration, reference material | Passive-Active |
| **Branching choice** | Select option, see consequences | Decision practice, judgment, scenario-based learning | Constructive-Interactive |
| **Sorting / ranking** | Order items by priority or sequence | Prioritization, process sequencing, triage | Active-Constructive |
| **Matching** | Connect related pairs | Association, terminology, cause-effect | Active |
| **Fill-in-the-blank** | Generate text (not select) | Recall, retrieval practice (stronger than recognition) | Constructive |
| **Slider / scale** | Position on a continuum | Self-assessment, estimation, calibration | Active |
| **Timeline** | Navigate events in sequence | Historical context, process development, project planning | Active |
| **Flashcard / flip** | Reveal hidden side | Retrieval practice, vocabulary, concept-definition pairs | Active |
| **Simulation / sandbox** | Manipulate a system and observe results | Systems thinking, troubleshooting, experimentation | Constructive-Interactive |

## Selection Principles

- **Match pattern to objective**: if the objective is "classify," use drag-and-drop or sorting. If the objective is "decide," use branching. If the objective is "recall," use fill-in-the-blank.
- **Interaction ≠ learning**: clicking an accordion is not the same as thinking. The interaction must require cognitive processing aligned to the objective.
- **Cognitive load**: complex interactions (multi-step drag-and-drop, nested branching) can overload novices. Start simple, build complexity.
- **Accessibility**: every interactive pattern must be keyboard-navigable and screen-reader compatible. Drag-and-drop is notoriously difficult to make accessible — always provide an alternative interaction mode.

## Common Mistakes

- **Click-to-reveal everything** — accordions and hotspots used as mandatory reading, not genuine exploration. If the learner must read everything, don't hide it.
- **Interaction as decoration** — adding drag-and-drop to content that doesn't need classification just to be "interactive"
- **No feedback** — interaction without feedback is activity without learning
- **Inaccessible interactions** — drag-and-drop and hotspots that don't work with keyboards or screen readers
