# Conceptual Visualization

## Summary

Conceptual visualization makes abstract ideas, relationships, and structures visible — not through data, but through spatial arrangement of concepts. These are the visuals used in strategy, teaching, design, and explanation: concept maps, frameworks, matrices, and systems maps.

## Types

| Type | Structure | Best for | Example |
|---|---|---|---|
| **Concept map** | Nodes + labeled links showing relationships | Showing how ideas relate; assessing mental models | "Memory → types → episodic + semantic + procedural" |
| **Mind map** | Central idea + radiating branches | Brainstorming, overview, quick capture | Topic exploration, planning |
| **Framework / model** | Named components in deliberate arrangement | Explaining a theory or system | "COM-B: Capability + Opportunity + Motivation → Behavior" |
| **2x2 matrix** | Two dimensions creating four quadrants | Classification, prioritization, trade-off analysis | "Urgency × Importance" (Eisenhower); "Evidence × Practice" |
| **Systems map** | Elements + feedback loops + causal arrows | Showing dynamics, unintended consequences, leverage points | "Low completion → manager skepticism → less time allocated → lower completion" |
| **Venn diagram** | Overlapping sets | Shared/unique attributes | Commonalities between two frameworks |
| **Hierarchy / tree** | Parent-child relationships | Taxonomy, org structure, classification | This knowledge base's layer structure |

## Key Principles

- **The spatial arrangement IS the argument** — where you place elements and how you connect them communicates relationships. Placement is not arbitrary.
- **Label the connections** — in concept maps, the links between nodes should be labeled ("causes," "enables," "requires," "contrasts with"). Unlabeled arrows are ambiguous.
- **Limit complexity** — a concept map with 50 nodes is unreadable. Show 5-9 elements per view. Use progressive disclosure for complexity.
- **Use consistently** — if you use a 2x2 matrix format once, use it again for comparable analyses. Consistency reduces cognitive load.
- **Test comprehension** — show the visual to someone unfamiliar with the content. Can they explain the main relationships? If not, simplify.

## Application to Learning Design

- **Schema elicitation** — have learners create concept maps to reveal their mental models ([Schema and Mental Models](../../principles/schema-and-mental-models.md))
- **Course overview** — a single framework visual that shows how all modules relate
- **Strategy communication** — 2x2 matrices and systems maps for executive audiences
- **Design documentation** — systems maps showing how layers of your learning design connect
