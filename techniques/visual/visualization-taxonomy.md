# Visualization Taxonomy

## Summary

Visualization is the broad practice of making information visible. Data visualization is a SUBSET — charts and graphs based on quantitative data. A learning designer needs BOTH: data visualization for evidence and evaluation, and broader visualization for explaining concepts, mapping processes, and telling stories.

## The Distinction

| Category | What it includes | Purpose |
|---|---|---|
| **Visualization (broad)** | Diagrams, maps, infographics, concept models, process flows, dashboards, spatial layouts, storyboards, frameworks | Explain, simplify, persuade, orient, reveal structure |
| **Data visualization (subset)** | Charts, graphs, tables, statistical plots, data-driven dashboards | Reveal patterns, compare values, show trends, show relationships, support decisions |

## Core Purposes of Visualization

Before choosing a visual form, know the JOB it must do:

| Purpose | Example forms |
|---|---|
| **Compare** | Bar chart, dot plot, table |
| **Show change over time** | Line chart, area chart, slope chart, timeline |
| **Show distribution** | Histogram, box plot, violin plot |
| **Show relationship** | Scatter plot, bubble chart, correlation matrix |
| **Show composition** | Stacked bar, treemap, waterfall |
| **Show hierarchy** | Tree diagram, treemap, sunburst, dendrogram |
| **Show flow** | Sankey diagram, alluvial chart, network graph, process map |
| **Show process** | Flowchart, swimlane, journey map, service blueprint |
| **Show geography** | Choropleth map, point map, heat map |
| **Tell a story** | Annotated chart, scrollytelling, infographic sequence |
| **Support monitoring** | Dashboard, KPI cards, alert indicators |
| **Explain a concept** | Concept map, framework, 2x2 matrix, systems map |

This is the first taxonomy a practitioner should know — it answers "what kind of visual do I need?" before "what chart do I make?"

## Non-Chart Visualization Types

| Type | Examples | Purpose |
|---|---|---|
| **Conceptual** | Concept maps, mind maps, frameworks, matrices, 2x2s, systems maps | Clarify ideas, show relationships, support strategy |
| **Process** | Flowcharts, swimlane diagrams, journey maps, service blueprints, operating models | Explain how something works |
| **Narrative** | Infographic sequences, scrollytelling, annotated charts, visual argument slides | Guide interpretation, not just display |
| **Interface/dashboard** | KPI cards, dashboards, filters, monitoring boards | Decision support, operational awareness |
| **Qualitative** | Thematic maps, affinity diagrams, coded matrices, stakeholder maps, quote clusters | Visualize themes, experiences, processes |

## The Selection Framework

**Data type + Task + Audience + Medium = Visual choice**

| Factor | Questions |
|---|---|
| **Data type** | Categorical? Ordinal? Numerical? Temporal? Geospatial? Hierarchical? Network? Textual? |
| **Task** | Compare? Trend? Distribute? Relate? Compose? Flow? Explain? Monitor? |
| **Audience** | Executives (top-line, simple)? Researchers (rigor, detail)? Learners (scaffolded, annotated)? Operations (real-time, thresholds)? |
| **Medium** | Slide? Report? Dashboard? Mobile? Print? Interactive? |
