# Typography for Learning Design

## Summary

Typography is how text looks and how that appearance affects readability, hierarchy, and processing. In learning materials, typography is a cognitive load tool — good typography makes text easy to process; bad typography makes every paragraph harder than it needs to be.

## Key Principles

### Readability first
- **Body text size**: 16px minimum on screen (14px absolute minimum). Smaller = reader fatigue.
- **Line length**: 45-75 characters per line is optimal. Too wide = eyes lose track; too narrow = choppy reading.
- **Line spacing**: 1.4-1.6x the font size for body text. Tight spacing creates visual density that slows reading.
- **Serif vs. sans-serif**: both are readable on screen with modern rendering. Sans-serif (e.g., Inter, Source Sans, Helvetica) is slightly conventional for digital; serif (e.g., Georgia, Charter) works well for long reading.

### Hierarchy through type
| Level | Treatment | Purpose |
|---|---|---|
| **H1** | Largest, boldest | Document/page title — only one per page |
| **H2** | Large, bold | Major sections — the structural skeleton |
| **H3** | Medium, bold or semi-bold | Subsections — one level of nesting |
| **Body** | Regular weight, comfortable size | The reading experience |
| **Caption/note** | Smaller, lighter | Secondary information |

### Font pairing
- **One font family is usually enough** — use weight (regular, bold) and size for hierarchy
- **If pairing**: one serif + one sans-serif. Use one for headings, one for body.
- **Never more than two font families** in a single design
- **Avoid decorative fonts** for anything other than a logo or a rare display element

## Common Mistakes

- **Too many fonts/weights/sizes** — creates visual noise instead of hierarchy
- **Center-aligned body text** — hard to scan; left-align is always safer
- **All caps for more than a few words** — reduces readability and feels like shouting
- **Low contrast text** — light gray on white is a readability killer (check WCAG contrast)
- **Justified text on screen** — creates uneven word spacing; left-align for screen

## How To Practice

1. Take a learning document and strip all formatting. Rebuild hierarchy using only: 2 sizes, 2 weights, and alignment. Does it work?
2. Check your body text: is it 16px+? Is line length under 75 characters? Is line spacing 1.4x+?
3. Screenshot a learning screen and squint: can you see the hierarchy from type alone?
