# Qualitative and Mixed-Method Visualization

## Summary

Qualitative visualization makes non-numerical information visible — interview themes, experiences, processes, relationships, and patterns in human behavior. Often overlooked in data visualization education (which focuses on charts), these formats are essential for learning designers working with learner research, stakeholder analysis, and experience design.

## Types

| Type | What it shows | When to use |
|---|---|---|
| **Affinity diagram** | Clustered themes from research data (interviews, observations, notes) | After qualitative data collection — organizing raw insights into themes |
| **Thematic map** | Visual arrangement of themes and their relationships | Showing how research themes connect and overlap |
| **Coded matrix display** | Themes × cases table with evidence summaries | Comparing how themes appear across different participants or contexts |
| **Stakeholder map** | People/groups positioned by influence, interest, relationship | Learner analysis, program planning, change management |
| **Empathy map** | What the user says, thinks, does, feels | Persona development, learner analysis, design thinking |
| **Journey map** | Experience over time with emotional arc | Learner experience, user experience (see [Process Visualization](process-visualization.md)) |
| **Systems map** | Elements, connections, and feedback loops | Understanding complex environments (see [Conceptual Visualization](conceptual-visualization.md)) |
| **Quote cluster** | Key quotes grouped by theme | Presenting qualitative evidence compellingly |
| **Evidence table** | Claims mapped to supporting evidence sources | Literature reviews, evaluation summaries |

## Key Principles

- **Qualitative visualization is INTERPRETIVE** — unlike a bar chart which represents objective counts, these visuals represent the analyst's interpretation of themes and patterns. Be transparent about the interpretive process.
- **Show the evidence trail** — link themes to specific data (quotes, observations, artifacts). This builds credibility and allows others to evaluate your interpretation.
- **Use spatial arrangement meaningfully** — proximity means relationship, size means importance, color means category. Same encoding principles as quantitative visualization.
- **Keep it revisable** — qualitative analysis is iterative. Use tools that allow rearrangement (sticky notes, Miro, Figma) rather than fixed formats.

## Application to Learning Design

- **Learner research** — affinity diagrams from interviews, empathy maps, stakeholder maps
- **Program evaluation** — thematic maps from focus groups, coded matrices from observation
- **Design documentation** — evidence tables linking design decisions to research findings
- **Research presentation** — quote clusters and thematic maps for communicating qualitative findings to stakeholders
