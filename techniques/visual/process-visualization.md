# Process Visualization

## Summary

Process visualization shows HOW THINGS WORK — steps, sequences, decisions, flows, and the relationships between actors and actions. These are among the most practically useful visuals in learning design, used for everything from instructional procedures to service blueprints to learner journeys.

## Types

| Type | Structure | Best for |
|---|---|---|
| **Flowchart** | Sequential steps with decision diamonds | Procedures, algorithms, troubleshooting guides |
| **Swimlane diagram** | Flowchart with horizontal/vertical lanes per actor | Multi-person processes, handoffs, role clarity |
| **Journey map** | Timeline-based experience from one person's perspective | Learner experience, user experience, emotional arc |
| **Service blueprint** | Multi-layer map (frontstage, backstage, support) | Full operational view of a learning program |
| **Operating model** | High-level view of how an organization functions | Program architecture, L&D function design |
| **Workflow diagram** | Task sequences with inputs/outputs | Production processes, content development workflows |
| **Decision tree** | Branching logic based on conditions | Assessment routing, learner pathway selection, troubleshooting |

## Key Principles

- **Direction matters** — left-to-right or top-to-bottom implies time flow or sequence. Be consistent.
- **Decisions are diamonds** — standard flowchart convention. Each decision has clear Yes/No or multiple branches.
- **One level of detail per diagram** — if a process has sub-processes, show the overview first, then drill into sub-process diagrams separately. Don't cram everything into one visual.
- **Label everything** — steps, decisions, arrows (what triggers the transition?), actors (in swimlanes). Unlabeled process diagrams are ambiguous.
- **Show the handoffs** — in multi-actor processes, the transitions between actors are where failures happen. Make them visible.
- **Journey maps include emotion** — unlike flowcharts, journey maps show how the person FEELS at each stage. This reveals pain points that process logic misses.

## Application to Learning Design

- **Learner pathway design** — decision trees for learner routing based on pretest results
- **Program operations** — service blueprints for blended programs
- **Procedure teaching** — flowcharts as job aids and reference materials
- **Design process documentation** — workflow diagrams for ADDIE/SAM phases
