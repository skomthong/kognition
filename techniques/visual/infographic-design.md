# Infographic Design

## Summary

An infographic is a visual communication piece that combines text, data, icons, and layout to explain a concept, tell a story, or summarize information in a scannable, memorable format. Unlike a chart (which shows data) or a slide (which supports a speaker), an infographic STANDS ALONE as a self-contained communication.

## Types

| Type | Structure | Best for |
|---|---|---|
| **Statistical** | Data-driven, charts + annotations + context | Summarizing research findings, making a data-driven argument |
| **Process/timeline** | Sequential steps or events | Explaining how something works, historical development |
| **Comparison** | Side-by-side or before/after | Showing differences between approaches, options, states |
| **Hierarchical** | Tree, pyramid, or layered structure | Showing organization, taxonomy, priority |
| **List-based** | Numbered or categorized items with visuals | Tips, principles, checklists, quick references |
| **Narrative** | Story-driven, guided sequence | Persuasive communication, case summary, journey visualization |

## Design Principles

### One message per infographic
Before designing, answer: "What is the ONE thing the viewer should take away?" Everything in the infographic serves that message. If you have three messages, make three infographics.

### Visual hierarchy guides the eye
The viewer should know where to start, where to go next, and what's most important — without instructions. Use size, color, position, and white space to create a clear reading path.

### Text is minimal and purposeful
- Headlines carry the message
- Body text supports with essential context
- Captions explain visuals
- If you need more than ~150 words, it's a document with illustrations, not an infographic

### Data must be honest
All chart and data visualization principles apply: accurate encoding, appropriate chart types, no truncated axes, no 3D distortion, sourced data.

### Accessibility
- Alt text for the entire infographic (summarize the message and key data)
- Sufficient contrast (WCAG)
- Don't rely on color alone for meaning
- Provide a text alternative for screen readers

## Common Mistakes

- **Too much information** — trying to fit an entire report into one graphic
- **Decoration over function** — pretty icons that don't carry meaning
- **No clear reading path** — viewer doesn't know where to start
- **Unsourced data** — numbers without attribution destroy credibility
- **Inaccessible** — no alt text, low contrast, meaning conveyed only through color

## How To Practice

1. Take a 500-word article and convert its key insight into a single infographic. Limit yourself to 50 words of text.
2. Design an infographic using only 2 colors + black + white. Notice how constraint forces hierarchy.
3. Show your infographic to someone for 10 seconds. Ask them: "What was the main message?" If they can't answer, redesign.
