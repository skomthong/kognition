# Slide Design

## Summary

Slides are the most common visual medium in learning design — and the most commonly abused. A slide is not a document, not a teleprompter, and not a decoration. A well-designed slide makes ONE point visually in a way that supports (not replaces) what the speaker or learner is doing.

## The Assertion-Evidence Model (Garr Reynolds / Michael Alley)

Replace the default "topic + bullet points" format:

| Traditional | Assertion-Evidence |
|---|---|
| **Title**: "Cognitive Load Theory" | **Assertion**: "Reducing extraneous load frees working memory for learning" |
| **Body**: 6 bullet points of text | **Evidence**: one diagram, image, or data visualization that supports the assertion |

The assertion IS the slide title — a complete sentence that states the point. The visual evidence supports it. No bullet points needed.

## Key Principles

### One idea per slide
If a slide makes two points, split it. Cognitive load applies to slides: each slide should be processable as a single chunk.

### Visual > text
- Use diagrams, charts, images, and process flows instead of text wherever possible ([Dual Coding](../../principles/dual-coding.md))
- If you must use text, use phrases — not sentences. The speaker provides the sentences.
- Never read slides aloud — that's the redundancy effect (CTML)

### Presenter deck vs. reader deck
| Type | Purpose | Design |
|---|---|---|
| **Presenter deck** | Supports a live presentation | Minimal text, strong visuals, speaker provides narration |
| **Reader deck** | Stands alone without a speaker | More text (but still structured), visual hierarchy, annotation |

Designing one deck for both purposes fails at both.

### Consistent visual system
- Same fonts, colors, and layout grid throughout
- Same treatment for the same type of content (all data slides look similar, all key-takeaway slides look similar)
- Template ≠ decoration — a template is a consistent visual system

## Common Mistakes

- **Wall of bullets** — 8 bullet points of full sentences. Reader can't process while listening.
- **Decorative templates** — gradients, swooshes, and logos that consume space without conveying information
- **Too much data** — a chart with 15 data series and no annotation. Highlight the ONE insight.
- **Reading the slides** — if you're reading slides aloud, the design has failed
- **Builds/animations for everything** — one strategic build can create emphasis; 20 builds create cognitive whiplash

## How To Practice

1. Take a bullet-point slide and convert it to assertion-evidence format. One sentence title + one supporting visual.
2. Design a 5-slide sequence where each slide makes exactly one point. Can someone flip through and get the argument from titles alone?
3. Design the same content as a presenter deck and a reader deck. Notice what changes.
