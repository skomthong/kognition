# Annotation Techniques

## Summary

Annotation is adding text, labels, callouts, and visual cues to a chart or diagram to guide the viewer's interpretation. A chart without annotation says "here's data." A chart WITH annotation says "here's what the data MEANS." Annotation is the bridge between visualization and [data storytelling](../narrative/data-storytelling.md).

## Types

| Type | What it does | Example |
|---|---|---|
| **Title as insight** | States the takeaway, not the topic | "Retention drops 60% without spacing" not "Retention Over Time" |
| **Callout/annotation** | Points to a specific data element and explains it | Arrow + text: "This is where the new program launched" |
| **Reference line** | Shows a benchmark, target, or threshold | Dashed line at 80% = "target completion rate" |
| **Highlighted element** | Uses color or weight to draw attention to one data point/series | Bold one series, gray the rest |
| **Direct labeling** | Puts values directly on the data instead of in a legend | Numbers on bar ends, labels on line endpoints |
| **Contextual note** | Explains methodology, caveats, or data source | "Based on N=247; excludes incomplete responses" |
| **Before/after marker** | Marks when an intervention occurred | Vertical line: "Program redesign launched here" |

## Key Principles

- **Annotate the insight, not the data** — "Revenue increased 23%" is a data label. "Revenue recovered after the pricing change, suggesting the intervention worked" is an insight annotation.
- **One primary annotation per visual** — the viewer should know what to look at FIRST. Multiple competing callouts create divided attention.
- **Direct label > legend** — every time a viewer looks from data to legend and back, they lose cognitive continuity. Put labels ON the data ([visual encoding](../../principles/visual-encoding.md) principle: reduce search).
- **Annotation = editorial judgment** — choosing what to annotate IS choosing what the viewer concludes. Be honest about this; frame transparently.
- **Less is more** — a chart buried in annotations is as bad as one with none. Annotate the single most important thing and let the rest breathe.

## How To Practice

1. Take a chart you've made with a generic title ("Q3 Results"). Replace the title with an insight statement.
2. Add one callout that points to the most important data element and explains WHY it matters.
3. Remove the legend and replace it with direct labels on the data. Is the chart clearer?
