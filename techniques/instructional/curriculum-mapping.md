# Curriculum Mapping

## Summary

Curriculum mapping is the process of aligning program-level outcomes, course/module-level objectives, assessments, and learning activities across an entire program — making the vertical (depth within a course) and horizontal (progression across courses) structure visible and auditable.

## How To Apply It

### Program-level alignment matrix

| Program outcome | Course 1 | Course 2 | Course 3 | Course 4 |
|---|---|---|---|---|
| Outcome A | Introduced | Practiced | — | Assessed at mastery |
| Outcome B | — | Introduced | Practiced | Assessed at mastery |
| Outcome C | Introduced | — | Practiced + Assessed | Assessed at mastery |

Use I (Introduced), P (Practiced), A (Assessed) or similar coding to show where each outcome is addressed.

### Course-level alignment table

| Module | Objectives | Assessment | Activities | Bloom level |
|---|---|---|---|---|
| Module 1 | Obj 1.1, 1.2 | Quiz + case analysis | Lecture, worked example, practice | Remember → Apply |
| Module 2 | Obj 2.1, 2.2 | Simulation rubric | Scenario practice, peer debrief | Apply → Analyze |

## Key Principles

- **Reveal gaps**: mapping often reveals that some program outcomes are never assessed, or that most assessment clusters at the recall level
- **Reveal redundancy**: mapping shows when the same content is taught in three courses but never scaffolded to a higher level
- **Sequence for progression**: use Bloom levels and complexity to plan a trajectory from foundational to advanced
- **Connect to competency frameworks**: for CBT programs, the curriculum map IS the competency framework implementation
- **Living document**: curriculum maps should be updated when courses change — treat as governance infrastructure, not a one-time exercise

## Cross-Module Integration Methods

Curriculum maps reveal what's taught WHERE. These techniques ensure modules connect into a coherent whole:

- **Bridging assignments**: tasks that require applying Module A's framework to Module B's content
- **Cumulative portfolios**: artifacts accumulate across modules; the portfolio IS the integration
- **Capstone projects**: final assessment that requires synthesizing knowledge from multiple modules
- **Spaced retrieval across modules**: Module 4 includes retrieval checks on Module 1-3 content (prevents siloed forgetting)
- **Shared case**: a single case or scenario that unfolds across modules, with each module contributing a new analytical lens
- **Cross-module reflection**: periodic "integration journals" where learners connect what they're learning across courses
- See [Spiral Curriculum](../../methods/spiral-curriculum.md) for within-program revisiting and [Longitudinal Learning Design](../../methods/longitudinal-learning-design.md) for multi-program progression

## How To Practice This Skill

1. Take an existing program (even a short one) and build the alignment matrix
2. Audit: are all program outcomes assessed? At what Bloom level? Is there progression?
3. Identify one gap and propose a redesign to address it
4. Compare the curriculum map to a learner journey map — where do they diverge?
